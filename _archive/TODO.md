# Uber BPS – Study List (Software Engineer II - L4)

**Interview structure:** ~45 min Coding + ~15 min Design & Architecture

---

## What to study

### Algorithms and data structures
- [ ] Common data structures: hash tables, trees, stacks, queues
	- [ ] Hash table script
	- [ ] Tree script
	- [ ] Stack script
	- [ ] Queue script
- [ ] Arrays, stacks/queues, hash tables, trees, graphs, heaps
- [ ] Runtime analysis (Big-O) and optimization techniques
- [ ] Problem-solving approaches common at large-scale tech companies

### Design & Architecture
- [ ] A project that shows complexity and scope of your role
- [ ] Architectural components and design decisions
- [ ] Trade-offs: why one technology was chosen over another
- [ ] Impact, scope, and results of past projects

### During the interview
- [ ] Pair programming / live coding
- [ ] Clarifying questions before coding
- [ ] Edge cases and mental code review
- [ ] Communicate your reasoning out loud
- [ ] Clean code, clear names, continuous testing
