# Assets — contexto de projeto (BPS / D&A)

Material de apoio para a parte de **Design & Arquitetura**: um projeto em que você teve **autoria** (decisões, ownership) e **impacto positivo** com o seu **know-how**.

## O que colocar aqui

- Arquivos em Markdown (`.md`), diagramas exportados, notas — o que ajudar a **estruturar a narrativa** sem depender só da memória na hora da entrevista.
- Foque em **papel seu**, **arquitetura**, **trade-offs**, **resultados** (qualidade, performance, escala, time, negócio), alinhado ao que a Uber costuma explorar na BPS (ver [`../README.md`](../README.md), seção D&A).

## Arquivo sugerido

- [`contexto-projeto.md`](contexto-projeto.md) — modelo com seções para preencher. Você pode duplicar para um **segundo** projeto ou renomear (ex.: `projeto-xyz.md`).

## Confidencialidade

Se o projeto for de cliente ou empresa, evite **dados sensíveis**, nomes internos desnecessários ou segredos comerciais; use generalizações quando precisar.
